NK Grid

This is a sample of the source code used to create this website.

Requirements and Considerations:
  • When building this website, I knew the content would be quite heavy since it's a portfolio. So, to reduce loading time, I wanted the code to add as little overhead as possible.
  • As the bulk of my content is images, I needed a mechanism to ease the process of maintenance and content management.
  • Once built, I wanted to be capable of modifying the content of my portfoliowithout touching the code.
Solution:
  • To reduce the code size, I decided to code in vanilla JavaScript, avoiding the use of any frameworks.
  • I started with atemplate from W3 Schoolas a foundation, as theirW3.CSS frameworkprovides a responsive UI driven purely by CSS. Although sample JS snippets are provided along with the CSS framework to demonstrate added functionality, no JS dependency is enforced, and any scripting required must be handled manually. This allowed me to write and repetitively prune any unrequired code, thus reducing the overhead of the download size.
  • To speed up content management, a config file containing details of the required images (location, description, the order of images, etc.) is loaded, which NK Grid then uses to dynamically render the content of the webpage at runtime. This architecture lets me simply modify the config file to reflect any changes or updates that I wish to make. Modifying the code is only required for implementing new features or enhancements.
Result:
The final script nkgrid.js occupies a meager7kbafter minification. The size of the config file, of course, cannot be determined, as it will vary based on the content of the page. Complete specs of this website after bundling and minification are given below:
  • JS:
    • Files: 3 scripts & 4 config files (at the time of writing)
    • Net size:18kb
  • CSS:
    • Files: 1 single CSS file (after bundling)
    • Size:49kb(includes W3.CSS framework)

iTunes Metadata Parser

This is a python script to read, parse, and extract data from an iTunes Library.

Files included in download:

  • Python Script
  • Sample iTunes Library XML file
Use Case & Requirements:

As a musician and an audiophile, I prefer to archive the majority of my music as uncompressed PCM data stored in either a WAV or AIFF container. Unfortunately, these containers do not support extended metadata, such as album or artist information. This is where iTunes comes in handy. When adding and manipulating such metadata within iTunes, the application tries to write the data directly on to the audio files. Most compressed formats like MP3 or AAC do support this feature. However, when dealing with WAV or AIFF, iTunes maintains it's own local database within which it stores such extended metadata. This is very useful for someone like me to organize my 200+ GB of music. The catch? Suppose my iTunes Library suddenly becomes corrupt. I have now lost all of that metadata and would be forced to spend countless hours painstakingly restoring it.

Luckily, iTunes has a built-in feature toexpose its library informationfor consumption by other third-party applications (perhaps to import your iTunes Library into other apps) as an XML file. This allows someone like me to jump right in and create a script to extract data from that file and store it as part of my archive. My preference is to have each folder (for example, Miles Davis's "Kind of Blue") be accompanied by another file containing the metadata related to the tracks within that folder. So, I wrote a python script that does just this; extract track metadata based on a given category (album, artist, etc) and, if required, a specific target (like the album "Kind of Blue"). I can then archive this along with the rest of my music for future use in music applications in the event of a data disaster.

Installation and Execution Instructions:
  1. Download and install Python 2.7.x (this script will not run on python 3.x). During installation, please be sure to allow python to execute from any directory. Suggested download link fromPython Software Foundation.
  2. Download and extract the source code for the iTunes Metadata Parser.
  3. Open up a terminal inside the directory containing the python script.
  4. Execute the following command:
                python itunes-xml-parser.py
